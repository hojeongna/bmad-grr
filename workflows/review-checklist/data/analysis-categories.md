# Analysis Categories

Categories and sub-areas to analyze when generating checklist items.
Used by project analysis agent and universal agent as a guide.

## Categories

### 1. Naming Conventions
- Component/class naming (PascalCase, camelCase, etc.)
- Variable/function naming patterns
- File naming conventions
- Constants naming (UPPER_SNAKE_CASE, etc.)
- Type/interface naming patterns

### 2. Folder Structure
- Directory organization patterns
- File placement rules (components, utils, hooks, etc.)
- Co-location patterns (tests, styles, types)
- Index file usage
- Module boundary conventions

### 3. Component Patterns
- Component structure and organization
- Props interface definitions
- State management patterns
- Event handler naming and placement
- Component composition patterns

### 4. Type Safety
- Type definitions location and organization
- Type vs interface usage
- Generic type patterns
- Type assertion rules
- Null/undefined handling

### 5. Import/Export Patterns
- Import ordering rules
- Barrel exports usage
- Relative vs absolute imports
- Circular dependency prevention
- Dynamic import patterns

### 6. State Management
- State management library usage
- Local vs global state decisions
- State update patterns
- Side effect handling
- Cache management

### 7. Error Handling
- Error boundary patterns
- Try-catch usage
- Error logging conventions
- User-facing error messages
- Fallback UI patterns

### 8. API & Data Flow
- API call patterns
- Data fetching conventions
- Request/response handling
- Data transformation location
- Loading/error state handling

### 9. Styling
- Styling approach (CSS modules, Tailwind, styled-components, etc.)
- Class naming conventions
- Design token usage
- Responsive design patterns
- Theme/dark mode handling

### 10. Testing
- Test file location and naming
- Test coverage expectations
- Testing library usage
- Mock patterns
- Test data management

### 11. Performance
- Memoization usage (useMemo, useCallback, React.memo)
- Bundle size considerations
- Lazy loading patterns
- Re-render prevention
- Asset optimization

### 12. Security
- Input validation
- XSS prevention
- Authentication/authorization patterns
- Sensitive data handling
- Environment variable usage

### 13. Constants & Configuration
- Constants definition location
- Magic number prevention
- Configuration management
- Environment-specific settings
- Feature flag patterns

### 14. Comments & Documentation
- JSDoc/TSDoc usage
- Inline comment conventions
- README requirements
- API documentation patterns
- Change log conventions
